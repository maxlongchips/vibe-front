# Prompt 库：前端专用咒语

> 好的 Prompt 不是"请帮我写代码"，而是精确的工程指令。

## 五层 Prompt 架构

来自 Stanford HAI 的研究：使用结构化 Prompt 模板的开发者，首次代码准确率比随意提问高 **60%**。没有结构化 Prompt 的团队，节省下来的时间有 40% 花在了返工上。

最有效的开发者 Prompt 遵循五层结构：

```
┌─────────────────────────────────────┐
│ 第 1 层：上下文                      │
│ 语言、框架、版本、架构模式、编码规范    │
├─────────────────────────────────────┤
│ 第 2 层：需求规格                     │
│ 具体的输入、输出、行为（不是"让它工作"）│
├─────────────────────────────────────┤
│ 第 3 层：约束和边界                   │
│ 性能要求、错误条件、边界情况           │
├─────────────────────────────────────┤
│ 第 4 层：集成点                      │
│ 这段代码如何连接到现有系统             │
├─────────────────────────────────────┤
│ 第 5 层：质量标准                     │
│ "完成"的定义是什么，测试期望           │
└─────────────────────────────────────┘
```

## Prompt 链式调用

复杂工作流不要塞进一个 Prompt。拆成链式调用：

```
步骤 1：分析需求 → 输出技术方案
步骤 2：基于方案生成实现代码
步骤 3：基于代码生成测试用例
步骤 4：审查代码的安全性和性能
```

每一步的输出是下一步的输入。这比一个巨大的 Prompt 效果好得多。

## Think 关键词

当任务需要深度推理时，使用关键词触发更多计算：

- `think` — 基础推理
- `think hard` — 深度分析
- `ultrathink` — 最大计算资源，用于架构决策
